I build agent **harnesses** — the unglamorous infrastructure around the model. Eval loops, tool governance, ratchets, HITL gates. The work that turns an LLM demo into a system you can run on a Tuesday and trust on a Friday.

### Currently shipping

- **[Nawneetverse](https://github.com/nawneet77)** &nbsp;·&nbsp; a 14-agent autonomous engineering loop. Runs every day, scores its own output, and ratchets the floor. Built on Claude + MCP + a custom orchestrator I keep rewriting until it gets boring.
- **[@Tracer-Cloud / opensre](https://github.com/Tracer-Cloud/opensre)** &nbsp;·&nbsp; merged [PR #1578](https://github.com/Tracer-Cloud/opensre/pull/1578) — a streaming guardrail adapter for their AI SRE agent. Open to lining up the next one.
- **Agent eval rubrics** &nbsp;·&nbsp; small, opinionated scoring frameworks I reuse across projects. The eval is the product.

### How I think about agents

- The model isn't the bottleneck. The harness around it is.
- Eval-driven beats vibe-driven. A ratchet beats a roadmap.
- An agent that ships once is a demo. An agent that ships every day is infrastructure.
- Human-in-the-loop is a feature, not a fallback.
